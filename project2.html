01. Project Overview

The ITHACA (Immersion and Training with Holograms for Art, Craft, and Audiovisual) project, funded by the European Union (No. 101104729), is a collaborative initiative involving multiple partners from research, industry, and creative fields. The project's goal is to make volumetric video technology more accessible and usable across various domains, including education, arts, and industry, by developing tools and workflows that simplify its creation, editing, and integration into Extended Reality (XR) applications.

As part of the ITHACA project, Fraunhofer is responsible for developing the Volumetric Video Editor and Extended Reality Viewer, with a particular focus on their application in industrial training scenarios.

  • Volumetric Video Editor: A tool that allows trainers to edit volumetric video content, add spatial annotations, overlay instructional audio, and structure training material into step-by-step chapters.
  • Extended Reality Viewer: An interactive XR application that enables trainees to view, manipulate, and integrate volumetric videos into their real-world environment, enhancing learning through immersive, hands-on experiences.

02. XR Viewer

The XR Viewer is an interactive application that allows users to view, manipulate, and interact with volumetric videos in an extended reality environment. It features an XR interface for navigation and tool selection within XR-based training scenarios.

Development Process

The Development of the XR Viewer started with research on VR UI best practices, followed by conceptualization, implementation, and iterative refinement based on user testing. The Development was organized using SCRUM.

Role

  • Research: Conducted initial UI and UX research, Analyzed existing VR menus and UI structures to inform interaction design.
  • Concept Design: Collaborated on wireframes and prototypes in Penpot.
  • Interaction Mapping: Designed and implemented intuitive button assignments for XR controllers, ensuring seamless interaction.
  • Implementation: Implemented the system responsible for UI functionality and interaction.

UI Components

  • Dashboard (Left Hand): Provides access to settings, environment selection, and navigation options.
  • Tool Selector (Right Hand): Enables switching between Transformation, Playback, and Filter tools.

Challenges

Prototyping XR interfaces is inherently complex, as traditional screen-based testing is not feasible. Ensuring usability and intuitiveness required multiple iterations and hands-on testing within the immersive environment.

03. XR Editor

The XR Editor is a tool that allows users to edit volumetric video content by adding spatial annotations, overlaying instructional audio, and structuring training material into well-organized trainings.

Development Process

To ensure an effective user experience, the XR Viewer was developed through a structured approach, starting with research on VR UI best practices, followed by conceptualization, implementation, and iterative refinement based on user testing.

User Research

To evaluate the usability of the XR Editor, we conducted a preliminary user study with a small number of participants. The study followed a think-aloud protocol during a guided use cases, where participants verbalized their thoughts while completing a predefined tasks within the XR environment. This approach helped us identify pain points and usability barriers in real-time.

Following the task completion, participants filled out the System Usability Scale (SUS) to quantitatively assess perceived usability. The results were analyzed using descriptive statistics and provided a benchmark for overall system acceptance. Additionally, qualitative feedback was collected through semi-structured interviews to uncover deeper insights and user needs.

Role

  • Research: Conducted initial UI and UX research, analyzed existing VR menus and UI structures to inform interaction design.
  • Interaction Mapping: Designed and implemented intuitive button assignments for XR controllers, ensuring seamless interaction.
  • Implementation: Implemented the system responsible for UI functionality and interaction.
  • User Research: Conceptualize, conduct and evaluate the user study, collaborated on a scientific paper about the Ithaca Editor (submitted and accepted to ICVR 2025).
